# Daily Menus — How These Work

Planning pattern for Plan C:

- **[itinerary.md](../itinerary.md)** defines the **locked skeleton** — time-sensitive bookings (teamLab, MiPig, Nintendo Museum lottery, Shinkansen, kaiseki) and travel legs. These are pinned to specific days.
- **Menus in this folder** are **flexible blocks** you can pick from day-of based on energy, weather, and mood. Each block is a self-contained unit: **activity + destination + food**.

### Block format

Every block has:
- **Area** — where it happens
- **Activity** — the thing to do (with ~duration)
- **Food** — a specific non-tourist-trap pick, type of cuisine, and why it's worth going
- **Vibe** — when this block lands best (active morning / slow afternoon / interesting evening)

### Food philosophy

We avoid tourist traps. Markers we look for:
- Small counters, single-dish specialists (ramen-only, tempura-only, unagi-only)
- Queues of Japanese salarymen at lunch
- Cash-only or ticket-machine entry
- Not on every English TripAdvisor list
- **Tabelog** (Japanese Yelp) rating 3.5+ is a strong signal; 3.8+ is excellent. Anything 4.0+ is destination-worthy.

We accept some "tourist-friendly but still authentic" places — Ichiran, Kura Sushi, Maisen — because they're genuinely good *and* kid/grandma friendly.
